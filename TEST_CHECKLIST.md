# ✅ 기능 테스트 체크리스트

프로젝트가 제대로 작동하는지 확인하기 위한 테스트 가이드입니다.

## 🔧 설치 확인

### 1. 환경 설정
- [ ] Python 3.8 이상 설치 확인
- [ ] 가상환경 생성 및 활성화
- [ ] `pip install -r requirements.txt` 실행
- [ ] `.env` 파일에 `OPENAI_API_KEY` 설정

### 2. 앱 실행
- [ ] `streamlit run app.py` 명령어 실행
- [ ] 브라우저가 자동으로 `http://localhost:8501` 열림
- [ ] 에러 없이 메인 페이지 로드됨

---

## 📊 기능 1: 히스토리 저장

### 테스트 시나리오
1. [ ] 메인 페이지에서 메뉴 추천 받기
2. [ ] "히스토리에 저장됨" 메시지 확인
3. [ ] 사이드바 > "📊 히스토리" 클릭
4. [ ] 방금 받은 추천이 목록에 표시되는지 확인
5. [ ] 통계 카드에 "총 추천 횟수" 업데이트 확인

### 추가 테스트
- [ ] 여러 번 추천 받아서 히스토리 누적 확인
- [ ] 히스토리 상세 정보 확장하여 내용 확인
- [ ] "히스토리 전체 삭제" 버튼 2번 클릭하여 삭제 확인
- [ ] 삭제 후 통계가 초기화되는지 확인

### 기대 결과
✅ 모든 추천이 자동으로 저장되고, 통계가 정확하게 표시됨

---

## 🌦️ 기능 2: 날씨 자동 반영

### 테스트 시나리오 (API 키 없이)
1. [ ] 메인 페이지 상단에 날씨 정보 표시되지 않음 (정상)
2. [ ] 추천은 정상 작동 (날씨 정보 제외)

### 테스트 시나리오 (API 키 있을 때)
1. [ ] `.env`에 `OPENWEATHER_API_KEY` 추가
2. [ ] 앱 재시작 (`Ctrl+C` 후 다시 실행)
3. [ ] 메인 페이지 상단에 현재 날씨 정보 표시 확인
   - "☀️ 맑음 | 15°C (체감 13°C)" 형태
4. [ ] "날씨 추천" 힌트 표시 확인
5. [ ] 메뉴 추천 받기
6. [ ] GPT 추천에 날씨 관련 언급이 있는지 확인
   - 예: "추운 날씨에는..."

### 기대 결과
✅ API 키가 있으면 날씨 반영, 없어도 정상 작동

## 🗳️ 기능 3: 그룹 투표

### 테스트 시나리오 - 투표 생성
1. [ ] 사이드바 > "🗳️ 그룹투표" 클릭
2. [ ] 이름 입력: "테스터"
3. [ ] 메뉴 3개 입력: "짜장면", "짬뽕", "탕수육"
4. [ ] "투표 생성" 버튼 클릭
5. [ ] 성공 메시지 및 공유 링크 생성 확인
6. [ ] 링크 복사

### 테스트 시나리오 - 투표 참여
1. [ ] 복사한 링크를 새 탭에 붙여넣기 (또는 시크릿 모드)
2. [ ] 이름 입력: "친구1"
3. [ ] "짜장면" 투표 버튼 클릭
4. [ ] "✅ '짜장면'에 투표했습니다!" 메시지 확인
5. [ ] 실시간 투표 현황에 "친구1" 표시 확인
6. [ ] 다시 원래 탭으로 돌아가서 투표 현황 확인

### 테스트 시나리오 - 투표 종료
1. [ ] 투표 생성자 화면에서 "투표 종료" 버튼 클릭
2. [ ] 투표 종료 메시지 확인
3. [ ] 최종 결과 표시 확인

### 기대 결과
✅ 실시간으로 투표가 동기화되고, 결과가 정확하게 표시됨

---

## 🗺️ 기능 4: 지도 연동

### 테스트 시나리오
1. [ ] 사이드바 > "🗺️ 주변음식점" 클릭
2. [ ] 메뉴 입력: "삼겹살"
3. [ ] 위치 입력: "강남"
4. [ ] 3개의 링크가 생성되는지 확인:
   - 네이버 지도
   - 카카오맵
   - 구글 검색
5. [ ] 각 링크 클릭하여 새 탭에서 올바르게 열리는지 확인
6. [ ] 검색 결과에 "강남 삼겹살"이 반영되는지 확인

### 인기 메뉴 버튼 테스트
1. [ ] "치킨" 버튼 클릭
2. [ ] 메뉴 입력란에 "치킨" 자동 입력 확인

### 기대 결과
✅ 모든 지도 링크가 올바르게 작동하고, 검색 결과가 정확함

---

## 📱 기능 5: 배달앱 연동

### 테스트 시나리오
1. [ ] 메인 페이지에서 메뉴 추천 받기
2. [ ] 추천 결과 하단의 "메뉴 선택 및 주문" 섹션 확인
3. [ ] 드롭다운에서 추천받은 메뉴 중 하나 선택
4. [ ] "선택한 메뉴: ..." 메시지 표시 확인
5. [ ] 위치 입력: "홍대"
6. [ ] 6개의 링크가 생성되는지 확인:
   - 배달의민족
   - 쿠팡이츠
   - 요기요
   - 네이버 지도
   - 카카오맵
   - 구글 검색
7. [ ] 배달의민족 링크 클릭하여 새 탭에서 검색 확인
8. [ ] 네이버 지도 링크 클릭하여 "홍대 [메뉴명]" 검색 확인

### 수동 입력 테스트
1. [ ] 메뉴 추출 실패 시 대체 입력란 표시 확인
2. [ ] 메뉴 직접 입력하여 링크 생성 확인

### 기대 결과
✅ 모든 배달앱 및 지도 링크가 올바르게 생성되고 작동함

## 🎯 통합 시나리오 테스트

### 시나리오: 점심 메뉴 정하고 주문하기

1. [ ] **메인 페이지** 접속
2. [ ] 현재 상태 입력:
   - 컨디션: 좋음
   - 스트레스: 5/10
   - 식사시간: 점심
   - 인원: 1명
   - 예산: 10,000원
   - 음식 종류: 한식
3. [ ] **날씨 확인**: 상단에 날씨 정보 표시
4. [ ] **AI 추천 받기** 버튼 클릭
5. [ ] 3가지 메뉴 추천 확인
6. [ ] **메뉴 선택**: 첫 번째 메뉴 선택
7. [ ] **배달의민족** 링크 클릭하여 주문 (실제 주문 X)
8. [ ] **히스토리** 페이지에서 방금 추천 확인
9. [ ] **통계** 확인: 총 1회, 평균 예산 10,000원

### 기대 결과
✅ 모든 기능이 순차적으로 연결되어 작동함

## ⚠️ 에러 처리 테스트

### API 키 없음
- [ ] `.env`에 OPENAI_API_KEY 없이 실행
- [ ] 친절한 에러 메시지 표시
- [ ] 앱이 크래시하지 않음

### 잘못된 입력
- [ ] 예산에 0원 입력 시 정상 작동
- [ ] 메뉴를 선택하지 않고 추천 받기 (기본값 작동)
- [ ] 투표 생성 시 메뉴 1개만 입력 (에러 메시지 표시)

### 네트워크 오류
- [ ] 인터넷 연결 없이 날씨 API 호출 (정상 작동, 날씨 정보만 표시 안됨)
- [ ] OpenAI API 오류 시 사용자 친화적 메시지
